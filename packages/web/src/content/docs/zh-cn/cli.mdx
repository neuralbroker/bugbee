Bugbee CLI 在不带任何参数运行时，默认启动 [TUI](/docs/tui)。

```bash
bugbee
```

但它也接受本页面中记录的命令，使您可以通过编程方式与 Bugbee 进行交互。

```bash
bugbee run "Explain how closures work in JavaScript"
```

---

### tui

启动 Bugbee 终端用户界面。

```bash
bugbee [project]
```

#### 标志

| 标志                                     | 简写 | 描述                                                      |
| ---------------------------------------- | ---- | --------------------------------------------------------- |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | 继续上一个会话                                            |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | 要继续的会话 ID                                           |
| <nobr><code>{"--fork"}</code></nobr>     |      | 继续时分叉会话（与 `--continue` 或 `--session` 配合使用） |
| <nobr><code>{"--prompt"}</code></nobr>   |      | 要使用的提示词                                            |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | 要使用的模型，格式为 provider/model                       |
| <nobr><code>{"--agent"}</code></nobr>    |      | 要使用的代理                                              |
| <nobr><code>{"--port"}</code></nobr>     |      | 监听端口                                                  |
| <nobr><code>{"--hostname"}</code></nobr> |      | 监听主机名                                                |

## 命令

Bugbee CLI 还提供以下命令。

---

### agent

管理 Bugbee 的代理。

```bash
bugbee agent [command]
```

---

### attach

将终端连接到已通过 `serve` 或 `web` 命令启动的 Bugbee 后端服务器。

```bash
bugbee attach [url]
```

这允许将 TUI 与远程 Bugbee 后端配合使用。例如：

```bash
# Start the backend server for web/mobile access
bugbee web --port 4096 --hostname 0.0.0.0

# In another terminal, attach the TUI to the running backend
bugbee attach http://10.20.30.40:4096
```

#### 标志

| 标志                                     | 简写 | 描述                                                                |
| ---------------------------------------- | ---- | ------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |      | 启动 TUI 的工作目录                                                 |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | 继续上一个会话                                                      |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | 要继续的会话 ID                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |      | 继续时派生会话（与 `--continue` 或 `--session` 一起使用）           |
| <nobr><code>{"--password"}</code></nobr> | `-p` | 基本认证密码（默认使用 `BUGBEE_SERVER_PASSWORD`）                 |
| <nobr><code>{"--username"}</code></nobr> | `-u` | 基本认证用户名（默认使用 `BUGBEE_SERVER_USERNAME` 或 `bugbee`） |

### auth

管理提供商的凭据和登录信息的命令。

```bash
bugbee auth [command]
```

---

#### login

Bugbee 基于 [Models.dev](https://models.dev) 的提供商列表运行，因此您可以使用 `bugbee auth login` 为任何想要使用的提供商配置 API 密钥。密钥存储在 `~/.local/share/bugbee/auth.json` 中。

```bash
bugbee auth login
```

Bugbee 启动时会从凭据文件加载提供商信息，同时也会加载环境变量或项目中 `.env` 文件中定义的密钥。

---

#### list

列出凭据文件中存储的所有已认证提供商。

```bash
bugbee auth list
```

或使用简写版本。

```bash
bugbee auth ls
```

---

#### logout

从凭据文件中清除提供商信息以完成登出。

```bash
bugbee auth logout
```

#### auth


您还可以列出支持 OAuth 的服务器及其认证状态。

```bash
bugbee mcp auth list
```

或使用简写版本。

```bash
bugbee mcp auth ls
```

### models

列出已配置提供商的所有可用模型。

```bash
bugbee models [provider]
```

此命令以 `provider/model` 的格式显示所有已配置提供商中可用的模型。

这对于确定在[配置文件](/docs/config/)中使用的确切模型名称非常有用。

您可以选择传入提供商 ID 来按提供商筛选模型。

```bash
bugbee models anthropic
```

#### 标志

| 标志                                    | 描述                                     |
| --------------------------------------- | ---------------------------------------- |
| <nobr><code>{"--refresh"}</code></nobr> | 从 models.dev 刷新模型缓存               |
| <nobr><code>{"--verbose"}</code></nobr> | 使用更详细的模型输出（包含费用等元数据） |

使用 `--refresh` 标志可以更新缓存的模型列表。当提供商新增了模型并且您希望在 Bugbee 中看到它们时，此功能非常有用。

```bash
bugbee models --refresh
```

### run

以非交互模式运行 Bugbee，直接传入提示词。

```bash
bugbee run [message..]
```

这对于脚本编写、自动化或无需启动完整 TUI 即可快速获取答案的场景非常有用。例如：

```bash "bugbee run"
bugbee run Explain the use of context in Go
```

您还可以连接到正在运行的 `bugbee serve` 实例，以避免每次运行时 MCP 服务器的冷启动时间：

```bash
# Start a headless server in one terminal
bugbee serve

# In another terminal, run commands that attach to it
bugbee run --attach http://localhost:4096 "Explain async/await in JavaScript"
```

#### 标志

| 标志                                     | 简写 | 描述                                                                |
| ---------------------------------------- | ---- | ------------------------------------------------------------------- |
| <nobr><code>{"--command"}</code></nobr>  |      | 要运行的命令，使用 message 作为参数                                 |
| <nobr><code>{"--continue"}</code></nobr> | `-c` | 继续上一个会话                                                      |
| <nobr><code>{"--session"}</code></nobr>  | `-s` | 要继续的会话 ID                                                     |
| <nobr><code>{"--fork"}</code></nobr>     |      | 继续时分叉会话（与 `--continue` 或 `--session` 配合使用）           |
| <nobr><code>{"--share"}</code></nobr>    |      | 分享会话                                                            |
| <nobr><code>{"--model"}</code></nobr>    | `-m` | 要使用的模型，格式为 provider/model                                 |
| <nobr><code>{"--agent"}</code></nobr>    |      | 要使用的代理                                                        |
| <nobr><code>{"--file"}</code></nobr>     | `-f` | 附加到消息的文件                                                    |
| <nobr><code>{"--format"}</code></nobr>   |      | 格式：default（格式化输出）或 json（原始 JSON 事件）                |
| <nobr><code>{"--title"}</code></nobr>    |      | 会话标题（未提供值时使用截断的提示词）                              |
| <nobr><code>{"--attach"}</code></nobr>   |      | 连接到正在运行的 bugbee 服务器（例如 http://localhost:4096）      |
| <nobr><code>{"--password"}</code></nobr> | `-p` | 基本认证密码（默认使用 `BUGBEE_SERVER_PASSWORD`）                 |
| <nobr><code>{"--username"}</code></nobr> | `-u` | 基本认证用户名（默认使用 `BUGBEE_SERVER_USERNAME` 或 `bugbee`） |
| <nobr><code>{"--dir"}</code></nobr>      |      | 运行目录，或附加时远程服务器上的路径                                |
| <nobr><code>{"--variant"}</code></nobr>  |      | 模型变体（特定于提供商的推理级别）                                  |
| <nobr><code>{"--thinking"}</code></nobr> |      | 显示思考块                                                          |
| <nobr><code>{"--port"}</code></nobr>     |      | 本地服务器端口（默认为随机端口）                                    |

### serve

启动无界面的 Bugbee 服务器以提供 API 访问。查看[服务器文档](/docs/server)了解完整的 HTTP 接口。

```bash
bugbee serve
```

此命令启动一个 HTTP 服务器，提供对 Bugbee 功能的 API 访问，无需 TUI 界面。设置 `BUGBEE_SERVER_PASSWORD` 可启用 HTTP 基本认证（用户名默认为 `bugbee`）。

#### 标志

| 标志                                     | 描述                       |
| ---------------------------------------- | -------------------------- |
| <nobr><code>{"--port"}</code></nobr>     | 监听端口                   |
| <nobr><code>{"--hostname"}</code></nobr> | 监听主机名                 |
| <nobr><code>{"--mdns"}</code></nobr>     | 启用 mDNS 发现             |
| <nobr><code>{"--cors"}</code></nobr>     | 允许 CORS 的额外浏览器来源 |

#### 标志

| 标志                                    | 描述                                                   |
| --------------------------------------- | ------------------------------------------------------ |
| <nobr><code>{"--days"}</code></nobr>    | 显示最近 N 天的统计信息（默认为所有时间）              |

| `BUGBEE_DISABLE_PRUNE`              | boolean | 禁用旧数据清理                          |
| `BUGBEE_DISABLE_TERMINAL_TITLE`     | boolean | 禁用自动终端标题更新                    |
| `BUGBEE_PERMISSION`                 | string  | 内联 JSON 权限配置                      |
| `BUGBEE_DISABLE_DEFAULT_PLUGINS`    | boolean | 禁用默认插件                            |
| `BUGBEE_DISABLE_LSP_DOWNLOAD`       | boolean | 禁用 LSP 服务器自动下载                 |
| `BUGBEE_ENABLE_EXPERIMENTAL_MODELS` | boolean | 启用实验性模型                          |

| `BUGBEE_DISABLE_CLAUDE_CODE`        | boolean | 禁用读取 `.claude`（提示词 + 技能）     |
